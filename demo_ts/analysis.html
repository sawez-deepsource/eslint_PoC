‚úÖ Master-Worker Wins

  • Fault isolation ‚Äî Worker OOM doesn't kill master
  • Recovery ‚Äî Automatic retry with smaller batches
  • Scalable ‚Äî Memory-gated spawning prevents cascade OOM
  • Visibility ‚Äî Per-worker memory timelines for debugging
  • Large repos ‚Äî Can handle repos that would OOM baseline

‚ùå Master-Worker Tradeoffs

  • Complexity ‚Äî More code, IPC, state management
